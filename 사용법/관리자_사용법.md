# eActive Resource Hub 관리자 사용 가이드

> 직원 공통 기능(로그인, 내 폴더, 문서 업로드, 설정 등)은 **직원 사용 가이드** 참조.
> 영업 기능은 **영업 사용 가이드** 참조.
> 이 문서는 관리자(ADMIN) 권한에서만 접근 가능한 기능을 설명합니다.

---

## 1. 관리자 대시보드

대시보드 → **관리자 패널** 클릭, 또는 상단 네비바 로고 클릭 (`/admin`).

### 통계 카드

- **전체 사용자**: 가입된 전체 계정 수. 클릭하면 직원 목록으로 이동
- **전체 팀**: 등록된 팀 수. 클릭하면 팀 관리로 이동
- **검토 대기**: 직원이 업로드한 문서 중 아직 검토하지 않은 건수. 클릭하면 문서 검토로 이동

### 빠른 메뉴

| 메뉴 | 기능 |
|------|------|
| 직원 관리 | 계정·권한·상태 관리 |
| 팀 관리 | 팀 생성·수정·삭제 |
| 문서 검토 | 대기 문서 승인·반려 |
| 만료 현황 | 기간 만료 문서 확인 |
| 통계 | 업로드·다운로드 활동 통계 |
| 재직증명서 | DOCX·PDF 자동 발급 |
| 양식·이력서 | 내 화면(직원 대시보드)으로 이동 |
| 인력표 팀 설정 | 팀별 인력표 노출 여부 관리 |
| 파일 GC | 고아 파일 정리 |
| 가입 허용 | 사전등록 이메일 관리 |

## 2. 직원 관리

관리자 대시보드 → **직원 관리** (`/admin/employees`).

### 검색 / 필터

이름 또는 이메일 키워드, 직급, 권한(관리자/영업/직원), 팀을 조합해 검색합니다.

### 직원 상세

목록에서 직원 행 클릭 시 상세 페이지로 이동합니다. 탭 6개로 구성됩니다.

#### 기본 정보 탭
이름, 이메일, 팀, 직급, 생년월일, 연락처, 역할, 계정 상태, 이메일 인증 여부, 폴더 생성 여부, 가입일을 확인합니다.

**계정 활성화 / 비활성화**
- 신규 가입자는 이메일 인증 완료 즉시 자동 활성화됩니다.
- 퇴직 등으로 접근을 차단할 때는 **비활성화** 버튼을 클릭합니다. 비활성화된 계정은 로그인이 차단되며 데이터는 유지됩니다.
- 관리자(ADMIN) 계정은 상태 변경이 불가합니다.

#### 문서 탭
해당 직원이 업로드한 전체 문서 목록이 표시됩니다. 제목·파일명으로 검색할 수 있으며, 미리보기·다운로드·검토 상태를 확인할 수 있습니다.

#### 역할 변경 탭
역할을 **직원(EMPLOYEE)** / **영업(SALES)** / **관리자(ADMIN)** 중 하나로 변경합니다. 변경 즉시 해당 직원의 세션이 만료되어 재로그인이 필요합니다.

#### 권한 관리 탭
특정 공유 폴더에 대한 개별 접근 권한을 부여하거나 회수합니다.

#### 직급 변경 탭
직급을 목록에서 선택해 변경합니다.

#### 팀 변경 탭
소속 팀을 목록에서 선택해 변경합니다. "팀 없음"으로 설정하면 미배정 상태가 됩니다.

## 3. 팀 관리

관리자 대시보드 → **팀 관리** (`/admin/teams`).

### 팀 생성
페이지 하단 생성 폼에 팀 이름을 입력하고 **추가** 버튼을 클릭합니다.

### 팀 수정
각 팀 카드의 **수정** 버튼 클릭 → 팀 이름 변경 → 저장.

### 팀 삭제
각 팀 카드의 **삭제** 버튼 클릭. 소속 직원이 있는 팀은 삭제되지 않으며, 직원을 먼저 다른 팀으로 이동해야 합니다.

---

## 4. 인력표 팀 설정

관리자 대시보드 → **인력표 팀 설정** (`/admin/teams/project-settings`).

인력표(`/sales/profiles`)에 표시할 팀을 켜고 끌 수 있습니다. 전사 인력 목록(`/sales/members`)에는 영향을 주지 않습니다.

- 초록 점(켜짐): 해당 팀이 인력표에 포함됨
- 회색 점(꺼짐): 해당 팀이 인력표에서 제외됨
- **끄기 / 켜기** 버튼으로 즉시 전환됩니다.

> 영업본부·경영본부는 기본값이 "꺼짐"입니다. 프로젝트 투입 가능한 팀만 켜두는 것을 권장합니다.

## 5. 문서 검토

관리자 대시보드 → **문서 검토** (`/admin/documents/review`).

직원이 업로드한 문서는 기본적으로 **검토 대기** 상태로 접수됩니다.

### 검토 목록

업로드 순으로 대기 문서가 나열됩니다. 문서 종류, 업로더, 업로드 날짜, 파일명이 표시됩니다.

### 개별 검토

목록에서 문서를 클릭하면 상세 검토 페이지로 이동합니다.

- **미리보기**: PDF·이미지 파일은 우측에서 바로 확인 가능합니다.
- **승인**: 문서가 정상이면 **승인** 버튼 클릭. 해당 직원의 폴더에 "승인됨" 배지가 표시됩니다.
- **반려**: 문제가 있으면 **반려** 버튼 클릭 → 반려 사유 입력 → 확인. 직원 폴더에 "반려됨" 배지와 사유가 노출되어 직원이 재업로드할 수 있습니다.

> 10MB 초과 파일은 관리자 검토 후 다운로드 가능 상태가 됩니다.

## 6. 만료 현황

관리자 대시보드 → **만료 현황** (`/admin/documents/expiry`).

만료일이 지났거나 30일 이내로 임박한 문서를 직원별로 확인합니다.

- **만료됨** 섹션: 이미 만료일이 경과한 문서 목록
- **만료임박** 섹션: 30일 이내 만료 예정인 문서 목록

직원에게 재발급을 요청해야 할 서류를 파악하는 데 사용합니다.

## 7. 재직증명서

관리자 대시보드 → **재직증명서** (`/admin/certificate`).

재직증명서 DOCX·PDF를 자동으로 생성합니다. 별도의 재직증명서 서비스(Python Flask)가 실행 중이어야 합니다. 서비스가 꺼져 있으면 상단에 안내 메시지가 표시됩니다.

### 단건 발급

드롭다운에서 직원 이름을 선택하고 **발급** 버튼 클릭. 해당 직원의 DOCX + PDF가 즉시 생성됩니다. 완료 후 결과(성공/실패 건수)가 표시됩니다.

### 전체 발급

**전체 발급** 버튼 클릭 → 확인 팝업 → 확인. 활성 직원 전원의 재직증명서를 일괄 생성합니다.

### 파일 목록

생성된 파일 목록이 표시됩니다. **최신** 배지가 표시된 파일이 가장 최근 생성본입니다. 파일명 클릭으로 다운로드합니다.

## 8. 통계

관리자 대시보드 → **통계** (`/admin/statistics`).

시스템 업로드·다운로드 활동 현황을 확인합니다.

- **다운로드 통계**: 전체·오늘·최근 7일·이번 달 건수
- **업로드 통계**: 전체·오늘·최근 7일·이번 달 건수
- **최다 다운로드 문서** 상위 10건
- **최근 다운로드 이력** 30건, **최근 업로드 이력** 20건

## 9. 파일 GC (Garbage Collection)

관리자 대시보드 네비바 → **파일 GC** (`/admin/gc`).

문서를 삭제해도 스토리지의 물리 파일은 즉시 삭제되지 않습니다. GC는 일정 보존 기간이 지난 삭제 문서의 실제 파일을 영구 제거합니다.

### GC 정책

| 항목 | 내용 |
|------|------|
| 보존 기간 | 기본값 (환경변수 `RESOURCEHUB_GC_RETENTION_DAYS`로 변경 가능) |
| 자동 실행 | 매일 새벽 02:00 자동 실행 |
| 삭제 대상 | 소프트 삭제(status=DELETED)된 지 보존 기간이 경과한 파일의 원본 + 미리보기 |

### 수동 실행

**지금 GC 실행** 버튼 클릭 → 확인 팝업 → 확인. 삭제 대상이 없으면 "정리 대상 없음"이, 처리된 경우 처리 건수가 표시됩니다.

> **주의**: 수동 GC로 삭제된 물리 파일은 복구할 수 없습니다.

## 10. 가입 허용 이메일 관리

관리자 대시보드 → **가입 허용** (`/admin/allowed-emails`).

회원가입이 가능한 이메일 주소를 사전 등록하고 관리합니다. 목록에 없는 이메일은 가입 폼 제출 시 차단됩니다.

### 이메일 추가

페이지 상단 입력창에 이메일 주소를 입력하고 **추가** 버튼 클릭. 이미 등록된 이메일은 오류 메시지가 표시됩니다.

### 이메일 삭제

목록에서 해당 이메일 오른쪽의 **삭제** 버튼 클릭. 삭제 후에는 해당 이메일로 신규 가입이 불가합니다 (기존 계정은 영향 없음).
