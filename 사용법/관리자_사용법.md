## 2. 최초 설정 — 시스템 처음 시작하기

시스템을 처음 도입할 때 필요한 설정 순서입니다.

### 1단계 — 관리자 계정으로 로그인

시스템 시작 시 자동으로 생성된 관리자 계정으로 로그인합니다:
- 이메일: `.env`의 `RESOURCEHUB_ADMIN_EMAIL` 값
- 비밀번호: `.env`의 `RESOURCEHUB_ADMIN_PASSWORD` 값

### 2단계 — 팀 생성

1. 상단 내비게이션 **관리자** → **팀 관리** 메뉴 이동
2. **새 팀 만들기** 버튼 클릭
3. 팀 이름과 설명 입력 후 저장
4. 사내 조직도에 맞게 팀을 모두 생성합니다

### 3단계 — 직원 계정 초대

직원이 스스로 회원가입하거나, 관리자가 직원에게 회원가입 링크를 안내합니다:
- 회원가입 URL: `http://[서버주소]/signup`
- 허용 도메인: 회사 이메일만 가입 가능 (`@eactive.co.kr`)

### 4단계 — 신규 직원 설정

가입한 직원에 대해 아래를 순서대로 설정합니다:
1. **역할 지정** (기본값: 사원 / 필요 시 영업 또는 관리자로 변경)
2. **팀 배정**
3. **개인 폴더 생성** (문서 업로드를 위해 필수)

## 4. 직원 관리

`/admin/employees` — **관리자** → **직원 관리**

### 직원 목록

- 이름, 이메일, 직급, 팀, 역할, 계정 상태(활성/비활성)가 표시됩니다.
- **검색**: 이름 또는 이메일로 검색
- **필터**: 팀, 직급, 역할별 필터
- **정렬**: 이름·팀·직급 기준 정렬
- **페이지네이션**: 20명씩 표시

### 직원 상세 — 탭 구성

직원 이름 클릭 시 상세 화면으로 이동합니다. 아래 탭이 있습니다:

#### [기본 정보 탭]

직원의 기본 정보와 다음 작업을 수행할 수 있습니다:

**계정 활성화 / 비활성화**
- **비활성화**: 해당 직원이 로그인 불가 상태로 전환됩니다. 비활성화 즉시 모든 활성 세션이 강제 만료됩니다.
- **활성화**: 다시 로그인 가능한 상태로 전환됩니다.
- 관리자 계정은 비활성화할 수 없습니다.

**개인 폴더 생성**
- 직원이 문서를 업로드하려면 개인 폴더가 필요합니다.
- 폴더가 없는 직원의 상세 화면에 **개인 폴더 생성** 버튼이 표시됩니다.
- 버튼을 클릭하면 즉시 폴더가 생성됩니다.

#### [팀 탭]

- 드롭다운에서 새 팀을 선택하고 **팀 변경** 버튼 클릭

#### [역할 탭]

변경 가능한 역할:
- **사원 (EMPLOYEE)**: 본인 폴더만 접근
- **영업 (SALES)**: 전 직원 열람 + 경력 계산기
- **관리자 (ADMIN)**: 전체 관리

**역할 변경 방법:**
1. [역할 탭] → **역할 변경** 버튼 클릭
2. 새 역할 선택 후 **변경** 클릭
3. 변경 즉시 해당 직원의 **모든 활성 세션이 강제 만료**됩니다. 해당 직원은 다시 로그인하면 새 권한으로 접속됩니다.

#### [문서 탭]

해당 직원이 업로드한 모든 문서 목록이 표시됩니다 (승인 여부 무관). 문서 클릭 시 상세 화면으로 이동합니다.

#### [권한 탭]

해당 직원에게 부여된 공유 폴더 접근 권한 목록이 표시됩니다. ([권한 관리](#8-권한-관리) 참고)

## 5. 문서 검토 및 승인

`/admin/documents/review` — **관리자** → **문서 검토**

파일 크기가 **10MB 이상**인 문서를 업로드하면 **검토 대기(PENDING_REVIEW)** 상태가 됩니다. 관리자가 검토 후 승인 또는 반려합니다. 10MB 미만 파일은 업로드 즉시 자동 승인됩니다.

### 검토 대기 목록

- 직원명, 문서 종류, 파일명, 업로드 일시가 표시됩니다.
- 목록 상단에 가장 오래된 대기 건이 표시됩니다 (선착순 처리 권장).

### 문서 검토 진행

1. 검토 목록에서 문서를 클릭합니다.
2. 우측 **미리보기** 영역에서 내용을 확인합니다.
   - PDF: 내장 뷰어로 바로 확인 가능
   - 이미지: 직접 표시
   - DOCX/HWP 등: 미리보기 없음, 다운로드 후 확인 필요
3. 검토 결과에 따라 버튼을 클릭합니다:

**승인:**
- **승인** 버튼 클릭
- 해당 문서가 **승인됨(APPROVED)** 상태로 변경됩니다.
- 문서 소유자에게 승인 알림 이메일이 자동 발송됩니다.
- 승인 후 영업 계정에서 해당 문서가 열람 가능해집니다.

**반려:**
- **반려** 버튼 클릭
- **반려 사유** 입력창에 사유를 2자 이상 입력합니다 (예: "파일이 손상되었습니다. 재업로드 부탁드립니다.").
- **반려 확인** 버튼 클릭
- 문서 소유자에게 반려 사유를 포함한 알림 이메일이 발송됩니다.
- 반려된 문서는 직원이 새 버전을 업로드하면 다시 검토 대기 상태로 올라옵니다.

> **주의**: 한 번 승인된 문서는 반려할 수 없습니다. 삭제가 필요하다면 [문서 강제 삭제](#11-직원-문서-직접-조회-및-삭제)를 사용하세요.

## 6. 문서 만료 관리

`/admin/documents/expiry` — **관리자** → **만료 현황**

### 만료 현황 화면

| 섹션 | 설명 |
|------|------|
| **만료된 문서** | 만료일이 이미 지난 문서 목록 |
| **만료 임박 문서** | 30일 이내 만료 예정 문서 목록 |

각 문서 항목에는 직원명, 문서 종류, 만료일이 표시됩니다.

### 만료 알림

시스템은 매일 정해진 시간에 자동으로 만료 확인을 수행하고:
- 만료일 당일: 문서 소유자에게 만료 알림 이메일 발송
- 만료 임박(30일 이내): 임박 알림 이메일 발송

> 이메일 미설정 시 콘솔 로그에만 출력됩니다.

## 7. 팀 관리

`/admin/teams` — **관리자** → **팀 관리**

### 팀 목록

등록된 모든 팀과 각 팀의 소속 인원 수가 표시됩니다.

### 팀 생성

1. **새 팀 만들기** 버튼 클릭
2. 팀 이름 (필수), 설명 (선택) 입력
3. **생성** 버튼 클릭

### 팀 수정

팀 목록에서 **수정** 버튼 클릭 → 이름과 설명 변경 가능

### 팀 삭제

**삭제** 버튼 클릭. 단, **소속 직원이 있는 팀은 삭제 불가**합니다. 먼저 소속 직원들을 다른 팀으로 이동한 후 삭제하세요.

## 8. 권한 관리

특정 직원에게 다른 직원의 개인 폴더 접근 권한을 부여합니다. 예: 팀장이 팀원들의 폴더를 열람해야 할 때.

### 권한 부여

**방법 1 — 직원 상세 화면에서:**
1. **직원 관리** → 해당 직원 클릭 → **[권한 탭]**
2. **권한 부여** 드롭다운에서 접근을 허용할 폴더(소유자명) 선택
3. **부여** 버튼 클릭

**방법 2 — 권한 관리 페이지에서:**
1. **직원 관리** → 해당 직원 → **역할 탭** → **권한 관리** 버튼
2. 동일하게 폴더 선택 후 부여

### 권한 회수

권한 목록에서 해당 항목의 **회수** 버튼을 클릭합니다. 즉시 적용됩니다.

### 주의사항

- 권한을 부여받은 직원은 **공유 폴더** 메뉴에서 해당 폴더를 볼 수 있습니다.
- 폴더 소유자 본인에게는 자신의 폴더 권한을 부여할 수 없습니다.
- 이미 부여된 권한을 중복으로 부여할 수 없습니다.

## 9. 이력서 템플릿 관리

`/admin/resume-template` — **관리자** → **이력서 템플릿**

직원들이 내 폴더에서 다운로드할 수 있는 이력서 양식 파일을 관리합니다.

### 템플릿 설정

1. **이력서 템플릿** 메뉴 이동
2. **템플릿 파일 업로드** — DOCX, HWP 등 이력서 양식 파일 선택
3. **활성화** 토글 — 활성화된 템플릿 파일 1개만 직원에게 제공됩니다
4. 직원 화면의 **내 폴더** 상단에 **이력서 양식 다운로드** 버튼이 표시됩니다

## 11. 직원 문서 직접 조회 및 삭제

관리자는 모든 직원의 문서를 직접 조회하고 삭제할 수 있습니다.

### 조회 방법 1 — 직원 상세 페이지

1. **직원 관리** → 해당 직원 클릭 → **[문서 탭]**
2. 해당 직원의 모든 문서 목록 확인
3. 문서 클릭 → 상세 화면 (미리보기, 버전 목록 포함)

### 조회 방법 2 — 문서 검색 (전체)

대시보드 검색 기능 또는 `/documents` 경로에서 전체 문서 검색이 가능합니다.

### 문서 강제 삭제

문서 상세 화면 하단 **문서 삭제** 버튼을 클릭합니다.

> **주의사항:**
> - 삭제 시 모든 버전과 파일이 **영구 삭제**됩니다.
> - 삭제 직전 확인 팝업이 표시됩니다.
> - 삭제 기록은 감사 로그에 남습니다.

## 12. 파일 GC 관리

`/admin/gc` — **관리자** → **파일 GC**

삭제된 문서의 물리 파일을 정리하는 Garbage Collection 기능입니다.

### 자동 정리

시스템은 **매일 새벽 2시** 자동으로 두 가지 정리 작업을 수행합니다:

| 작업 | 대상 | 기준 |
|------|------|------|
| **소프트 삭제 문서 정리** | 삭제 처리된 문서의 물리 파일 | 삭제 후 7일 경과 |
| **고아 파일 정리** | DB에 기록이 없는 스토리지 파일 | 생성 후 1시간 경과 |

### 수동 실행

디스크 정리가 급하게 필요한 경우 수동으로 실행할 수 있습니다.

1. **파일 GC** 메뉴 이동
2. 현재 정리 대기 건수 확인
3. **GC 실행** 버튼 클릭
4. 완료 후 처리 건수가 표시됩니다.

> 자동 정리가 매일 실행되므로 평소에는 수동 실행이 필요 없습니다.

## 13. 세션 및 계정 보안

### 세션 만료

- 세션은 **30분 비활동** 시 자동 만료됩니다.
- 만료 후에는 로그인 화면으로 리다이렉트됩니다.

### 직원 계정 강제 세션 만료

직원을 비활성화하면 해당 직원의 **모든 활성 세션이 즉시 강제 만료**됩니다. 퇴사 처리 또는 계정 잠금 시 즉각 효력이 발생합니다.

### 감사 로그

시스템은 다음 작업을 자동으로 감사 로그에 기록합니다:
- 문서 업로드 / 삭제
- 문서 승인 / 반려

## 14. 자주 묻는 질문

**Q. 직원이 가입했는데 개인 폴더가 없다고 합니다.**  
A. 개인 폴더는 자동 생성되지 않습니다. **직원 관리** → 해당 직원 상세 → **[기본 정보 탭]** → **개인 폴더 생성** 버튼을 클릭해 수동으로 만들어주세요.

**Q. 회사 이메일 도메인을 변경하고 싶어요.**  
A. 서버의 `.env` 파일에서 `RESOURCEHUB_COMPANY_EMAIL_DOMAIN` 값을 변경한 후 애플리케이션을 재시작합니다. 변경 후 새 도메인 이메일로만 회원가입이 가능합니다.

**Q. 관리자 비밀번호를 분실했어요.**  
A. 로그인 화면의 **비밀번호를 잊으셨나요?** 기능으로 재설정하거나, 데이터베이스에서 직접 비밀번호를 업데이트해야 합니다:
```sql
-- BCrypt 해시값으로 업데이트 (예: Admin@2024! 의 해시)
UPDATE users SET password = '$2a$10$...' WHERE email = 'admin@eactive.co.kr';
```
BCrypt 해시는 온라인 도구나 Spring Security의 `BCryptPasswordEncoder`로 생성합니다.

**Q. 직원을 삭제하고 싶은데 삭제 기능이 없어요.**  
A. 직원 계정은 삭제하지 않고 **비활성화** 처리합니다. 비활성화된 계정은 로그인이 불가능하며, 데이터는 보존됩니다. 이는 문서·감사 로그 등 관련 데이터의 무결성을 유지하기 위한 정책입니다.

**Q. 팀을 삭제하려는데 "소속 직원이 있어 삭제 불가" 오류가 납니다.**  
A. 해당 팀 소속 직원들을 먼저 다른 팀으로 이동한 후 삭제하세요. **직원 관리** → 각 직원 → **[팀 탭]** → 새 팀 선택 → 팀 변경.

**Q. 승인한 문서를 되돌리고 싶어요 (재검토).**  
A. 승인을 취소하는 직접적인 기능은 없습니다. 문서를 삭제하거나, 직원에게 새 버전을 업로드하도록 요청하세요. 새 버전 업로드 시 다시 검토 대기 상태가 됩니다.

**Q. 특정 직원이 로그인 중인지 확인할 수 있나요?**  
A. 현재 세션 목록 조회 기능은 없습니다. 계정 비활성화를 통해 강제 로그아웃은 가능합니다.

**Q. 파일 업로드 용량 제한을 늘리고 싶어요.**  
A. `src/main/resources/application.yml`에서 수정합니다:
```yaml
spring:
  servlet:
    multipart:
      max-file-size: 50MB      # 파일 하나 최대 크기
      max-request-size: 100MB  # 요청 전체 최대 크기
```
수정 후 애플리케이션을 재시작합니다.
