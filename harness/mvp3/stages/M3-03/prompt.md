# MVP3 M3-03 — Prompt

MVP3 3단계.

작업: `/signup` 폼 정비.

요구사항:

1. 입력 필드 순서를 다음과 같이:
   - 이름 → 생년월일 → 연락처 → 회사 이메일(앞부분) → 팀 → 직급 → 비밀번호 → 비밀번호 확인

2. 생년월일 입력을 8자 텍스트로 변경
   - input type="text", pattern="\\d{8}", maxlength=8
   - placeholder "20010904"
   - 서버에서 YYYYMMDD → LocalDate.parse 변환 (DateTimeFormatter.ofPattern("yyyyMMdd"))
   - 검증: 1900-01-01 ~ 오늘 사이
   - 사용자 입력 형식 오류 시 한글 메시지 ("생년월일은 8자 숫자로 입력하세요. 예: 20010904")

3. 직급 select 에 상무 노출
   - M3-01 에서 enum/displayName 갱신됐으니 select 자동 반영. 누락된 옵션 없는지 확인.

4. 팀 select
   - 기존 teams 테이블의 팀 목록을 옵션으로 노출 (이미 있을 가능성)
   - 가입 직후 사용자가 팀을 선택 → users.team_id 에 저장

5. 이메일 잘못 입력 시 비밀번호 유지
   - 폼 검증 실패 시 model 에 입력값 다시 채워주는 패턴이 mvp2 03 부터 있을 것 — 회귀 안 깨지게.
   - 비밀번호와 비밀번호 확인은 보안상 다시 입력해야 하는 게 일반적이지만, 사용자 요청은 "유지" — 입력값을 model 에 다시 노출.
   - 단, 콘솔/로그/응답에 평문 비번 출력 금지.

6. NOT-DOING
   - /dashboard 변경
   - DocumentType
   - 태그

검증:
- /signup 화면 입력 순서 확인
- 생년월일 "20010904" 입력 → 가입 성공
- 생년월일 "2001-09-04" 입력 → 거부 (8자 숫자 아님)
- 생년월일 "20011231"(미래) 입력 → 정책에 맞게 거부 또는 통과
- 직급 select 에 상무 노출
- 이메일에 일부러 오타 입력 + 비번 입력 후 제출 → 거부 시 비번 필드에 입력값 유지
- DB에 저장된 birth_date 가 LocalDate
- mvp1·mvp2 기존 가입 흐름 그대로 동작
