# 사용자 시나리오

> 역할별 핵심 사용 흐름. 테스트 케이스 설계와 QA 체크리스트의 기준으로 사용.
> 참고 소스: `docs/spec.md`, `docs/frontend.md`, 각 Controller/Service

---

## 관리자(ADMIN) 시나리오

### A-1. 새 직원 계정 확인 및 활성화 (이메일 인증 후 추가 검토 없음)
1. 직원이 `/signup`에서 회원가입 완료 → 이메일 인증 → `ACTIVE` 자동 전환
2. 관리자가 `/admin/employees`에서 신규 계정 목록 확인
3. 필요 시 직급·팀·역할 수정 (`/admin/employees/{userId}` 탭별로)
4. 부적합 계정은 toggle-status로 `DISABLED` 처리

### A-2. 문서 검토 (승인)
1. 직원이 문서 업로드 → `PENDING_REVIEW` 상태
2. 관리자 대시보드의 "검토 대기" 카드 수치 확인
3. `/admin/documents/review` → 대기 문서 목록
4. 문서 클릭 → 미리보기 확인 → 승인 버튼 클릭
5. `APPROVED` 전환 → 영업/권한자 접근 가능

### A-3. 문서 검토 (반려)
1. A-2의 1~3단계 동일
2. 반려 버튼 클릭 → 사유 입력 필수 → 반려 제출
3. `REJECTED` 전환 → 직원에게 재업로드 요청

### A-4. 만료 문서 확인
1. `/admin/documents/expiry` 접속
2. 만료된 문서 목록과 임박 문서 목록 확인
3. 필요 시 해당 직원에게 갱신 요청

### A-5. 직원 폴더 접근 권한 부여
1. `/admin/employees/{userId}` → 권한 탭
2. 부여 가능한 폴더 목록 → 대상 폴더 선택 → 부여
3. 해당 직원이 `/shared/folders`에서 해당 폴더 열람 가능

### A-6. 팀 관리
1. `/admin/teams` → 팀 생성 또는 수정
2. `/admin/teams/project-settings` → 영업 인력표 노출 여부 토글

### A-7. 재직증명서 발급
1. `/admin/certificate` → 직원 선택
2. 생성 버튼 → Python Flask 컨테이너 호출 → DOCX·PDF 생성
3. 다운로드

### A-8. 파일 GC 수동 실행
1. `/admin/gc` → 보존 기간 확인
2. "GC 실행" 버튼 → soft-delete 후 1시간 이상 경과 파일 삭제
3. 처리 건수 결과 메시지 확인

### A-9. 공용 폴더 파일 업로드
1. `/shared/folders/public` → 업로드 버튼 (ADMIN만 표시)
2. 파일 선택·제목 입력 → 업로드
3. 전 직원이 해당 파일 다운로드 가능

## 영업(SALES) 시나리오

### S-1. 인력표 조회 및 엑셀 내보내기
1. `/sales/profiles` 접속
2. 필터 설정(팀·직급·가용 상태·기술등급 등)
3. 원하는 컬럼 선택·정렬
4. 대상 사원 체크박스 선택
5. 엑셀 내보내기 → xlsx 다운로드

### S-2. 프리셋 저장 및 재사용
1. S-1의 1~3단계 — 원하는 컬럼·정렬 세팅
2. "프리셋 저장" → 이름 입력
3. 다음 접속 시 프리셋 선택으로 빠른 복원

### S-3. 번들 다운로드
1. `/sales/profiles` → 사원 다수 체크
2. 번들 다운로드 → 이력서+경력 문서 ZIP 파일 다운로드

### S-4. 경력 계산기 사용 및 저장
1. `/sales/career-calculator` 접속
2. 자동채움으로 기존 구간 불러오거나 직접 날짜 구간 입력
3. 중복 제거 여부 선택 → 계산
4. 결과 확인(Y년 N개월) → 저장 → EmployeeProfile 업데이트

### S-5. 직원 문서 개별 조회
1. `/sales/members` → 직원명 클릭
2. APPROVED 문서 목록 → 파일 다운로드

## 사원(EMPLOYEE) 시나리오

### E-1. 회원가입
1. `/signup` → 이름·생년월일·연락처·이메일·직급·(팀)·비밀번호 입력
2. 가입 신청 → 이메일 인증 코드 수신
3. `/signup/verify` → 코드 입력 → ACTIVE + 개인 폴더 자동 생성

### E-2. 문서 업로드
1. `/my/folder` → 업로드 버튼
2. 문서 유형·제목·파일 선택, 만료일·발급일(선택) 입력
3. 업로드 완료 → `PENDING_REVIEW` 상태
4. 관리자 승인 후 영업/권한자 접근 가능

### E-3. 문서 삭제
1. `/my/folder/documents/{documentId}` → 삭제 버튼
2. soft-delete 처리 → 목록에서 사라짐
3. 파일은 GC(매일 02:00)에서 실제 삭제

### E-4. 공용 폴더 파일 다운로드
1. `/shared/folders/public` 접속
2. 파일 목록 확인 → 다운로드

### E-5. 비밀번호 변경
1. `/settings?tab=password`
2. 현재 비밀번호 입력 → 새 비밀번호 입력(복잡도 조건) → 저장

### E-6. 비밀번호 찾기
1. `/login/forgot` → 이메일 입력
2. 6자리 코드 이메일 수신
3. `/login/forgot/verify` → 코드 입력(5분 이내) → 새 비밀번호 설정
4. `/login`으로 리다이렉트

### E-7. 이력서 템플릿 다운로드
1. `/dashboard` 또는 `/my/folder` → 이력서 템플릿 메뉴
2. `GET /my/folder/resume-template/download` → 관리자 업로드 최신 DOCX 다운로드

### E-8. 권한 부여된 폴더 열람
1. 관리자가 A-5로 접근 권한 부여
2. `/shared/folders` → 해당 폴더 목록 확인
3. 폴더 클릭 → APPROVED 문서 열람·다운로드

## 경계 시나리오 (에러 케이스)

| 시나리오 | 예상 동작 |
|----------|----------|
| 미인증 상태로 `/my/folder` 접근 | `/login`으로 리다이렉트 |
| EMPLOYEE가 `/admin/**` 접근 | 403 에러 페이지 |
| SALES가 `/admin/**` 접근 | 403 에러 페이지 |
| 권한 없는 개인 폴더 문서 다운로드 URL 직접 입력 | 403 응답 |
| PENDING_REVIEW 버전 타인이 다운로드 URL 직접 입력 | 403 응답 |
| 만료된 인증 코드 입력 | 오류 메시지, 재시도 유도 |
| 이미 가입된 이메일로 회원가입 | 오류 메시지 |
| 동일 폴더에 같은 타입·제목 문서 재업로드 | 오류 메시지 (V211 unique constraint) |
| 허용되지 않은 확장자 업로드 | 오류 메시지 |
| magic bytes 불일치 파일 업로드 | 오류 메시지 |
| DISABLED 계정으로 로그인 시도 | 로그인 실패 |
